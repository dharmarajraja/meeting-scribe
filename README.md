# Meeting Scribe

Desktop app (Electron + React + TypeScript) that records a meeting, produces a
live, speaker-labeled transcript, and generates structured meeting minutes
with Mistral when you stop recording.

Two tabs:
- **Transcript** — live, timestamped, speaker-labeled, searchable, exportable to `.docx`.
- **Meeting Minutes** — executive summary, key decisions, action items (task/owner/due date), open questions, and next steps, generated by Mistral from the transcript, exportable to `.docx`.

## Architecture

- **Audio capture**: microphone (`getUserMedia`) + system/meeting audio (`getDisplayMedia` via Electron's loopback `setDisplayMediaRequestHandler`, Windows), mixed into one stream with the Web Audio API so Deepgram's diarization covers both you and remote participants.
- **Transcription**: [Deepgram](https://deepgram.com/) live streaming API (`nova-2` model, diarization + interim results), streamed from the renderer to the Electron main process over IPC, then to Deepgram — the API key never touches the renderer.
- **Summarization**: [Mistral](https://console.mistral.ai/) (`mistral-large-latest`, JSON mode), called from the main process once recording stops, given the full final transcript, and prompted to return structured JSON (summary, decisions, action items, open questions, next steps).
- **Export**: `.docx` generation via the `docx` library, saved through a native OS save dialog.

## Setup

1. **Install dependencies**
   ```
   npm install
   ```

2. **Add your API keys** — copy `.env.example` to `.env` and fill in:
   - `DEEPGRAM_API_KEY` — from https://console.deepgram.com/
   - `MISTRAL_API_KEY` — from https://console.mistral.ai/
   - Never commit `.env` (already in `.gitignore`).

3. **Check the Mistral model ID** in `src/main/mistral.ts` (`DEFAULT_MODEL`) against the current model list in the [Mistral docs](https://docs.mistral.ai/getting-started/models/) — model IDs and capability tiers change over time.

4. **Run in development**
   ```
   npm run dev
   ```

5. **Build a Windows installer**
   ```
   npm run build:win
   ```

## Using the app

1. Enter a meeting title.
2. Click **Start Recording**. The OS will prompt for microphone access, and Electron's screen/audio-share picker will appear for system audio — pick the screen/window whose audio you want captured (e.g., your meeting app window). If you decline system audio, the app falls back to mic-only.
3. Watch the **Transcript** tab fill in live as people speak.
4. Click **Stop Recording**. The app automatically generates the **Meeting Minutes** tab from the final transcript.
5. Export either tab to `.docx` for sharing.

## Known limitations / next steps

- **Speaker labels are numeric** (`Speaker 1`, `Speaker 2`, …) from Deepgram's diarization — there's no name-mapping UI yet. A natural next step: let the user rename speakers after the fact, and have that rename apply retroactively to the transcript and re-generated minutes.
- **System audio loopback capture is Windows-focused** (Electron's `audio: 'loopback'` request handler). macOS/Linux system-audio capture works differently and isn't wired up yet.
- **No persistence layer yet** — transcripts/minutes live only in memory for the current session until exported. Adding local storage (SQLite) of past meetings is a reasonable Phase 2.
- **Recording-consent banner is not yet implemented** — add an on-screen notice before capture starts if this will be used with external participants, per standard meeting-recording consent practice.
- **Table/view counts, cost, and error-rate benchmarks are not yet measured** for this specific setup — validate Deepgram accuracy and Mistral minutes quality against a real meeting recording before relying on it for anything business-critical.
